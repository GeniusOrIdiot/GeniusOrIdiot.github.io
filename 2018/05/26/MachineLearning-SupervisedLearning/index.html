这种数量的样本实在不足以说明什么问题，也实在无法给出具有建树性的意见，我查阅了最近3年当地中学的录取线，然后根据这张图的走势（整体还是上升的趋势）和平均值，给了一点建议。

分析问题

我们先分析一下，这个问题，目的有两个：预测分数、决定志愿

预测分数/名次

上述案例中的数据集Data Set可以说是很没有参考性了，能预测的东西很有限。对于给出的数据集，需要一个输入input和一个输出output，输入可以是影响到考试的因素，如试题难度、考试状态等，输出则比较直观，就是成绩/名次，尤为重要的一点，就是数据量要大，要很大，否则，偶然性因素Accidental Factor会极大干扰预测结果。

像下面这样一个数据集(部分)，可以抽象出来的信息就比较多了：

相对来说，这样的图就比较正常logical，成绩稳定的人，这样的线一般更趋近于水平线，而受难易度影响较大的则会显示出滑坡decline的趋势。

相对来说，中考这样的大型考试，难度会比较中庸，姑且认为是在6附近，那么可以大致预测该考生的中考成绩会在如图575分上下。当然，这也仅仅是预测predict而已，万一人家一个不小心超常发挥了呢？🤣

回到机器学习

问题似乎已经分析得很明了了，但这和机器学习又有什么关系呢？

聪明的小伙伴也许已经能领会到，对上面的例子中的图表，进行拟合、分析、得出结论的，难道是人类自己吗？不！少量的数据量，或许可以自己人工manual处理，但是如果是一个孩子一学期的成绩考核汇总、或是整个学校的学生报考志愿系统的话，那个数据量绝不是人类可以胜任的！

所以，机器学习Machine Learning是必不可少的。

上述的两个问题，正是机器学习中监督学习Supervised Learning的两类问题：回归问题Regression Problem和分类问题Classification Problem。

监督学习：程序需要一定成本的数据集对数据进行分析、拟合，从而对相关的问题给出符合规律的结果或结论。(不要被名称吓到，其实就是机器从数据中总结规律，给出建议的学习方式)

两种问题最根本的区别在于：学习算法最终得到的是一个连续可预测的结果值回归问题，还是一个划分类别的判定值分类问题。

回归问题

正如上述预测分数/名次的问题，我们需要根据数据集Data Set 对以后每种难度的试题预测考生的分数，这样就需要对散点图进行拟合，形成一条符合规律的曲线或是直线，实际上是将数据集里的离散值进行回归，得到回归函数，从而对函数上任意位置的结果进行预测（计算）。

其实早在高中阶段的数学课程中就有接触过这类问题——线性回归Linear Regression，回归问题里较为简单的一种。（(#`O′)！高中的东西谁还记得啊！🙄）

那我们更早其实在初中就有接触过这类问题——一次函数又称线性函数Linear Function。（高中都记不得，还提初中？！😤————不对不对，一次函数好歹还记得一点……🤔）

举个栗子吧~

小明去买汽水，第一次他只买了一瓶，给了老板3块钱，后来一次体育课，他帮同学带，一共要买5瓶，他不知道自己钱带的够不够，希望在座的各位大佬帮忙算一下……😏

别小看这个问题哦~这可是线性回归问题的鼻祖(认真脸~)！

首先，你不要管它什么 总价=单价*数量，万一这个老板是个不良商家，第二瓶价格翻倍怎么办？又或者老板其实是个好人Good Man，第二瓶半价，第三瓶再半价！那又怎么办？

所以说总价=单价*数量只是更符合规律regular，才被认为是正确的算法Algorithm。假设总价为y，数量为x，那么这个线性函数就是：$ y=3*x $

然后，我们就基于这个算法使用了几千年~

现在我们跳跃一点，看这个函数：

$$ y=θ_1x_1+θ_2x_2+θ_3x_3+… $$

和上面的 $ y=3*x $ 比起来也就多了一堆类似 $ 3 * x $ 之类的东西罢了嘛~

你大可以理解为：

小明现在是去超市买东西，买了很多很多东西：汽水 $ x_1 $ 瓶、雪糕 $ x_2 $ 根、薯片 $ x_3 $ 袋……

受每样东西单价的影响，以及购买数量的约束，总价会不一样。

只要小明往超市跑的次数足够多，给的样本数足够多，通过线性回归的算法，是可以计算每种商品的单价的——当然，是在你不直接去超市询问价格的前提下虽然没人这么做~🙃。

这之后，只要小明说他要买哪些东西，各要多少，你就可以给他算出总价，避免他因为少带了钱而尴尬。

实际上，线性回归只是回归问题的一种，我们需要掌握和理解的是这样一种算法的概念和想法idea，至于那些个 $ θ_1、θ_2、θ_3…$ 怎么算，管它呢！反正有工具帮我们算。

分类问题

Pro.Ng课上介绍了判断肿瘤是良性还是恶性的问题，我还想引出一个更有意思的话题——判断一个人是男，是女——还是伪娘？

这里首先要搞清楚一个细节：这个分类，不是说必须要得出是/否的结论，只要得出的结论是一个类别区分的结果集就可以了~比如说：男、女、伪娘😏

Pro.Ng从年龄和肿瘤的大小两个维度将数据集Data Set进行了整理，最终得到一个聚类形式的散点图：

红色代表恶性malignant，蓝色代表良性benign，这个算法会根据这个图，画一条直线把数据集分开，如果现在有一个病人，年龄和肿瘤大小对应的点在直线下方，这个算法就会判断病人的肿瘤是良性的。

这是一个有两个特征Features的分类问题，但实际上判断肿瘤的性质需要的绝不仅仅只有两个维度，为了便于理解，Pro.Ng才这么讲解。

对于性别，涉及到的特征也有很多，比如说：头发长短、是否有胡子、说话的音调、是否有体香等等，以此作出的聚类图会是一个多维的图(还不会用Octave画图真的对不住)。

那伪娘算什么呢？看一下上面的图，良性肿瘤聚集区是不是有一个恶性的？这个肿瘤它喜欢模仿良性肿瘤的一些特征，从而达到鱼目混珠的目的。所以，你选择的维度越多，特征就越明显，最终的分类也会更加精确。

监督学习Supervised Learning

什么样的事情需要监督呢？

比如考试，最后你需要上交一份试题的解答；比如说工作，最终你要做完领导安排给你的事情……

受监督的事情最终都要有一个特定Specific的结果。预测分数、判断志愿达成、预测肿瘤性质、预测性别……这些都需要给出一个明确的答案answer。

相对的，就会有一些事情，不需要给出特别明确的结果的。比如网易云的智能推荐，它收集你听过的歌作为数据集，并没有对此给出一个特定的结论，它只是对你的品味进行了整理归纳，然后给你推荐类似的歌曲而已。

这种学习算法我们称之为：非监督学习Unsupervised Learning
